Project Requirements Document consists of all the conditions and tasks which are mentioned by the client that must be completed to ensure the completion of the project. It will further include class diagrams, high level architecture of the project and mock ups.
title
Once the we freeze the requirements, we use Jira for planning and controlling of our development cycle. Every requirement will be assigned a user story and every user story will be assigned one or multiple tasks. Time estimations will be made and any kind of deviations will be recorded.
title
We start hacking. Each and every small PR is reviewed and the following things are checked- it’s up to the standard we need, covers all edge cases, has proper documentation, there are no hacks and the code does not create technical debts.Before the code review we have CI that runs linting checks, integration tests, unit tests and functional tests in parallel.
title
Lastly we check for code coverage with test coverare to make sure it’s 80% at least. Once all the check passes it goes to staging server. Every PR is deployed in an environment that is close to production.After this based on the test metrics decided in the task it goes to Q/A. Once approved by Q/A, we merge to master. On merge to master our CircleCI build triggers again and deploys to production
